Enterprise Data Governance & GDPR/NDPA Compliance Framework

Problem: Utility organisations manage sensitive customer and operational data, but often lack governance aligned with NDPA and audit expectations.

Approach: Developed a comprehensive framework covering data classification, RBAC, audit logging, compliance workflows, and risk management tailored to utility operations.

Technologies & Methods: Governance models, GDPR compliance mapping, access control design, audit and risk controls.

Outcome: Improved regulatory readiness, reduced data-handling risk, and established a scalable governance blueprint.

Key challenges addressed
  • Unclear ownership of data domains and inconsistent handling of sensitive data
  • Limited auditability (who accessed what, when, and why)
  • Need for practical controls that fit utility operations (metering, billing, customer service)
Deliverables
  • Data classification & handling standard (PII/PSI tagging)
  • RBAC model + access review process
  • Audit logging & monitoring design
  • Risk register template and compliance checklist

Big Data Application Migration (On‑Prem → Cloud)

Problem: On‑prem warehouses were expensive, rigid, and hard to scale for growth.

Approach: Designed a phased batch migration strategy with cutover planning to minimise downtime while optimising cost/performance.

Technologies: AWS / Azure / GCP building blocks (storage, compute), migration tooling, batch orchestration concepts.

Outcome: Improved scalability and resilience, reduced infrastructure overhead, and enabled a cloud‑ready analytics foundation.

Key challenges addressed
  • Reducing cutover risk while migrating large volumes of data
  • Preserving data quality, lineage, and security controls during transition
  • Balancing cloud cost vs performance (right-sizing and storage strategy)
What I delivered
  • Migration runbook (phases, dependencies, rollback plan)
  • Validation checks (row counts, reconciliation, sampling, schema drift)
  • Target architecture blueprint + operating model for the cloud
