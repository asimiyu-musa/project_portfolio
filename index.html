Enterprise Data Governance & NDPA Compliance Framework

Problem: Utility organisations manage sensitive customer and operational data, but often lack governance aligned with NDPA and audit expectations.

Approach: Developed a comprehensive framework covering data classification, RBAC, audit logging, compliance workflows, and risk management tailored to utility operations.

Technologies & Methods: Governance models, NDPA compliance mapping, access control design, audit and risk controls.

Outcome: Improved regulatory readiness, reduced data-handling risk, and established a scalable governance blueprint.

Big Data Application Migration (On‑Prem → Cloud)

Problem: On‑prem warehouses were expensive, rigid, and hard to scale for growth.

Approach: Designed a phased batch migration strategy with cutover planning to minimise downtime while optimising cost/performance.

Technologies: AWS / Azure / GCP building blocks (storage, compute), migration tooling, batch orchestration concepts.

Outcome: Improved scalability and resilience, reduced infrastructure overhead, and enabled a cloud‑ready analytics foundation.

Big Data Processing & Quality Assurance Pipeline

Problem: Inconsistent data quality created unreliable reporting and poor decision confidence.

Approach: Implemented profiling, quality checks, and automated validation rules embedded within processing workflows.

Technologies: Python, SQL, automated data testing concepts, quality metrics.

Outcome: Increased trust in datasets, reduced downstream failures, and standardised quality controls.

Auto‑Intel: Automated Data Intelligence Platform

Problem: Manual collection and analysis slowed market intelligence and reduced insight timeliness.

Approach: Built an automated platform integrating scraping, validation, transformation, orchestration, API delivery, NLP insights, and dashboards.

Technologies: Python, Scrapy, PostgreSQL, Apache Airflow, NLP, Streamlit / BI tooling.

Outcome: End‑to‑end automation improved insight velocity, reliability, and scalability for analytics users.

End‑to‑End Streaming Pipeline (Kafka & Airflow)

Problem: Batch processing couldn’t meet near real-time analytics requirements.

Approach: Engineered a streaming pipeline for ingestion and processing of high-volume events, prepared for visualisation and reporting.

Technologies: Apache Kafka, Apache Airflow, Python.

Outcome: Enabled timely analytics and operational responsiveness with scalable event-driven architecture.

Real Estate Property Records ETL (PostgreSQL)

Problem: Property records were siloed and hard to query consistently.

Approach: Built an ETL pipeline to standardise, transform, and load property datasets into PostgreSQL with analytics-friendly structure.

Technologies: Python, PostgreSQL, SQL optimisation.

Outcome: Streamlined access, faster reporting, and improved data usability for downstream applications.
